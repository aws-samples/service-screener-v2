# Kinesis Data Streams Simulation Testing

Scripts to create two intentionally-suboptimal Kinesis Data Streams so
every `kinesis*` service-screener check that can be forced through the
AWS API within seconds is validated end-to-end.

## Resources Created

All prefixed with `ss-test-`:

| Resource | Configuration | Directly Validates |
|---|---|---|
| Kinesis stream `ss-test-kinesis-basic-*` | ON_DEMAND, `EncryptionType=NONE`, no tags, `RetentionPeriodHours=24`, no shard-level metrics, no alarms | `kinesisSSEDisabled`, `kinesisRetentionPeriodMinimum`, `kinesisNoTags`, `kinesisEnhancedMonitoringDisabled`, `kinesisNoCloudWatchAlarms` |
| Kinesis stream `ss-test-kinesis-provisioned-*` | PROVISIONED (1 shard), `EncryptionType=KMS` with `KeyId=alias/aws/kinesis`, only `IncomingBytes` metric enabled, no tags, retention 24h, no alarms | `kinesisSSEDefaultKey`, `kinesisProvisionedModeNoAutoScaling`, `kinesisEnhancedMonitoringPartial`, `kinesisNoTags`, `kinesisRetentionPeriodMinimum`, `kinesisNoCloudWatchAlarms` |

## Coverage

| # | Check | Simulated? |
|---:|---|---|
| 1 | `kinesisSSEDisabled` | ✓ FAIL (basic stream) |
| 2 | `kinesisSSEDefaultKey` | ✓ FAIL (provisioned stream) |
| 3 | `kinesisKMSKeyRotationDisabled` | ✗ N/A — `alias/aws/kinesis` skips this check by design. Requires a customer-managed CMK without rotation to force. |
| 4 | `kinesisRetentionPeriodMinimum` | ✓ FAIL (both streams — default is 24h) |
| 5 | `kinesisNoTags` | ✓ FAIL (both streams) |
| 6 | `kinesisEnhancedMonitoringDisabled` | ✓ FAIL (basic stream) |
| 7 | `kinesisStreamNotActive` | ✗ Transient — only fires while a stream is in CREATING/UPDATING/DELETING. May briefly appear if the scanner runs mid-creation. |
| 8 | `kinesisProvisionedModeNoAutoScaling` | ✓ FAIL (provisioned stream) |
| 9 | `kinesisShardUtilizationHigh` | ✗ Requires > 80% of account shard quota — impractical to force. |
| 10 | `kinesisNoConsumers` | ✗ PASS (stream is < 7 days old → check is deferred). |
| 11 | `kinesisOnDemandPredictableWorkload` | ✗ PASS (stream is < 30 days old → check is deferred). |
| 12 | `kinesisEnhancedMonitoringPartial` | ✓ FAIL (provisioned stream — only `IncomingBytes` enabled, missing critical metrics) |
| 13 | `kinesisConsumerStuck` | ✗ Requires an EFO consumer stuck > 10 min — hard to force reliably. |
| 14 (spec #22) | `kinesisNoCloudWatchAlarms` | ✓ FAIL (both streams — no alarms configured) |

**Directly forced to FAIL: 8 of 14** (checks 1, 2, 4, 5, 6, 8, 12, 14).
**Correctly PASSING with expected reason: 4 of 14** (checks 3, 10, 11, 13).
**Impractical to force: 2 of 14** (checks 7, 9).

## Cost

Effectively **cents**:
- Basic stream (ON_DEMAND, no traffic): billed per-hour + PUT payload units. Zero PUT payload → hours only.
- Provisioned stream (1 shard, no traffic): ~$0.015/hour = ~$0.015 per hour of testing.
- No PutRecord traffic is generated by these scripts.

Delete streams within an hour of testing to keep the bill under $0.05.

## Usage

```bash
cd services/kinesis/simulation
chmod +x create_test_resources.sh cleanup_test_resources.sh
./create_test_resources.sh --region ap-southeast-1

# Wait ~30s for encryption to activate on the provisioned stream, then:
cd ../../..
python3 main.py --regions ap-southeast-1 --services kinesis --beta 1 --sequential 1

cd services/kinesis/simulation
./cleanup_test_resources.sh --force
```

## IAM Permissions Required

**For the create/cleanup scripts:**
- `kinesis:CreateStream`, `kinesis:DeleteStream`, `kinesis:DescribeStreamSummary`,
  `kinesis:StartStreamEncryption`, `kinesis:EnableEnhancedMonitoring`
- `sts:GetCallerIdentity`

**For the screener (already documented in the main service):**
- `kinesis:ListStreams`, `kinesis:DescribeStreamSummary`,
  `kinesis:ListTagsForStream`, `kinesis:ListStreamConsumers`,
  `kinesis:DescribeLimits`
- `kms:GetKeyRotationStatus` (for check #3)
- `cloudwatch:DescribeAlarmsForMetric` (for check #14)

## Notes

- **AWS-managed key rotation**: When the provisioned stream is encrypted
  with `alias/aws/kinesis`, `kinesisKMSKeyRotationDisabled` returns "not
  applicable" — AWS rotates managed keys automatically and customers
  cannot control it. To force check #3, replace the alias with a
  customer-managed CMK that has `EnableKeyRotation=false`.
- **Encryption timing**: `StartStreamEncryption` returns immediately but
  the stream stays in `UPDATING` for ~30-60 seconds. Running the screener
  during this window may briefly trigger `kinesisStreamNotActive` — this
  is expected and self-heals.
- **Alarm coverage**: The `kinesisNoCloudWatchAlarms` check queries
  `AWS/Kinesis` metrics with a `StreamName` dimension. Alarms defined at
  the account level (no dimensions) will not be counted.
