# Cognito Simulation Testing

Scripts to create an intentionally-insecure Cognito user pool (plus one app
client with long-lived tokens) to validate the 12 `cognito*` service-screener
checks.

## Resources Created

All prefixed with `ss-test-`:

| Resource | Configuration | Directly Validates |
|---|---|---|
| User pool `ss-test-cognito-pool-*` | `MfaConfiguration=OFF`, password policy `MinimumLength=6` with only lowercase required, no `UserPoolAddOns`, `DeletionProtection=INACTIVE`, `AutoVerifiedAttributes=[]`, no `LambdaConfig`, `AccountRecoverySetting` with **one** mechanism (`verified_email`), no `DeviceConfiguration`, no `UserPoolTags`, and 0 users | #1, #2, #3, #4, #5, #6, #8, #9, #11, #12 |
| App client `ss-test-cognito-client-*` | `AccessTokenValidity=24h`, `IdTokenValidity=24h`, `RefreshTokenValidity=365d` | #10 |

## Cost

Effectively **$0** — Cognito user pools on the LITE tier are billed by
monthly active users. A pool with zero sign-ins costs nothing.

## Usage

```bash
cd services/cognito/simulation
chmod +x create_test_resources.sh cleanup_test_resources.sh
./create_test_resources.sh
# Cognito is immediately consistent — no propagation wait needed.
sleep 30
cd ../../..
python3 main.py --regions us-east-1 --services cognito --beta 1 --sequential 1
cd services/cognito/simulation
./cleanup_test_resources.sh --force
```

## IAM Permissions Required

- `cognito-idp:CreateUserPool`, `cognito-idp:DeleteUserPool`,
  `cognito-idp:UpdateUserPool`, `cognito-idp:DescribeUserPool`,
  `cognito-idp:ListUserPools`
- `cognito-idp:CreateUserPoolClient`, `cognito-idp:DeleteUserPoolClient`,
  `cognito-idp:DescribeUserPoolClient`, `cognito-idp:ListUserPoolClients`
- `sts:GetCallerIdentity`

## Notes

- Cognito user pools are region-scoped; the scripts default to
  `us-east-1` (override with `--region`).
- The IDs generated by `create-user-pool` cannot be recreated after
  deletion. The cleanup script explicitly disables deletion protection
  before delete in case the pool was manually flipped to `ACTIVE` during
  testing.
- App clients are deleted implicitly with the pool, but the cleanup script
  deletes them explicitly first for clarity in the output.
